# Contributing

Please follow these guidelines when contributing:

- Use feature branches: `feature/<short-description>`.
- Commit messages: present-tense, short summary, optional body.
- Open a Pull Request to `main` (or `backend/main`) describing the change and linking related issues.
- Ensure `.env` secrets are not committed.

Pull request checklist

- [ ] Tests (if any) pass
- [ ] Linter (if present) passes
- [ ] No secrets in changes
- [ ] PR description and screenshots (if UI)
